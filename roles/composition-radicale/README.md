# Radicale

[Radicale](https://radicale.org) is a lightweight, self-hosted CalDAV/CardDAV server.
Used here as the storage backend for [`composition-gateway`](../composition-gateway)'s
Reminders (CalDAV `VTODO` collections) and Contacts (CardDAV addressbook) tool groups.

**Radicale sits behind Gateway, not in front of it.** Gateway is the only CalDAV/CardDAV
client of this instance — the `gw` CLI and a companion iOS app talk plain JSON to
Gateway's `/mcp` tools and `/v1` API, never CalDAV/CardDAV directly. (An earlier attempt
had a third-party iOS app, [NowThis](https://nowthis.app/), speak CalDAV to Radicale
directly; no iOS task app does CalDAV-synced geofencing, which is why that approach was
abandoned in favour of Gateway as a translation layer.) See the gateway repo's
[`docs/superpowers/specs/2026-07-19-radicale-contacts-reminders-design.md`](https://github.com/awfulwoman/gateway/blob/main/docs/superpowers/specs/2026-07-19-radicale-contacts-reminders-design.md)
for the full design.

## Ports

Internal port `5232`. Exposed via Traefik at `radicale.<domain>`.

## Environment

Configured entirely via `RADICALE_CONFIG_*` environment variables (no mounted config file):

- Storage backend: filesystem, at `/data/collections`
- Auth: `htpasswd` with bcrypt, file at `/config/users`

## Volumes

| Path | Purpose |
|------|---------|
| `{{ composition_config }}/collections` | CalDAV/CardDAV collections (reminder lists + the contacts addressbook) |
| `{{ composition_config }}/users` | htpasswd auth file, generated by the `Create htpasswd file` task from vault-stored credentials |

## DNS

Registers subdomain: `radicale`
