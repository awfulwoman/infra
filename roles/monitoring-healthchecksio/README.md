# Monitoring Healthchecks.io

Registers a host with [Healthchecks.io](https://healthchecks.io) and sends a success ping during each playbook run, providing dead-man's-switch style monitoring for infrastructure automation.

The check is created (or confirmed to exist) via the Healthchecks.io API, tagged with the host type and inventory name for easy filtering. A success ping is then sent to confirm the playbook ran successfully. If the check stops being pinged — because a host is down or Ansible stops running — Healthchecks.io sends an alert.

## Key Variables

| Variable | Default | Description |
|---|---|---|
| `monitoring_healthchecksio` | `true` | Enable the role |
| `monitoring_healthchecksio_name` | `{{ inventory_hostname }} - Core Playbook` | Name of the check in Healthchecks.io |
| `monitoring_healthchecksio_debug` | `false` | Print API response details |
| `vault_healthchecks_rw_apikey` | _(required)_ | Read-write API key from Healthchecks.io, stored in Ansible Vault |

## Design Notes

- The check schedule is hardcoded to `0 8 * * *` (daily at 08:00 UTC), matching the expected frequency of the `automation-infra` playbook runs.
- Checks are matched by name (`unique: ["name"]`) so re-running the role does not create duplicate checks.
- The entire block is wrapped in a `rescue` that silently handles the case where the `community.healthchecksio` collection is not installed — the playbook continues rather than failing. This makes the role safe to include broadly without requiring the collection on every control node.
- Tags (`autogenerated`, host type, hostname) make it easy to group and filter checks in the Healthchecks.io dashboard.
