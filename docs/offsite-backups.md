# Offsite Backups

Offsite backups protect against site-level failure: fire, theft, or total hardware loss. The offsite backup strategy is intentionally separate from the onsite ZFS replication system.

## What gets backed up offsite

Offsite backup coverage follows the ZFS policy system:

| Policy | Offsite |
|---|---|
| `none` | No |
| `low` | No |
| `high` | **Yes** |
| `critical` | **Yes** |

See [ZFS Architecture](ZFS.md) for policy definitions and how to assign policies to datasets.

## Technology

Offsite backups use [Borg](https://borgbackup.readthedocs.io/) to a [Hetzner Storage Box](https://www.hetzner.com/storage/storage-box/).

- **Encryption**: `repokey-blake2` — key stored in repository, protected by a per-host passphrase
- **Compression**: lz4
- **Deduplication**: Borg's content-addressed chunking — incremental backups are efficient
- **Transport**: SSH on port 23
- **Schedule**: Daily at 02:30 (configurable per host)

Borg runs directly on each source host, not via the onsite backup server. This means offsite backups are independent of the `server-8gb-backups` onsite replication chain.

## Ansible role

The `backups-borg-client` role installs and configures Borg on a host. Apply it to any host with ZFS datasets that have `policy: high` or `policy: critical`. See the [role README](../roles/backups-borg-client/README.md) for the full variable reference and Borg command cheatsheet.

Minimal host_vars to add:

```yaml
borg_storage_box_host: u123456.your-storagebox.de
borg_storage_box_user: u123456
vault_borg_passphrase: !vault |
  $ANSIBLE_VAULT;1.2;AES256;beanpod
  ...
```

Generate the passphrase:

```bash
ansible-vault encrypt_string "$(openssl rand -hex 32)" --name 'vault_borg_passphrase'
```

Add the role to the host's `core.yaml` playbook:

```yaml
roles:
  - backups-borg-client
```

## Hetzner Storage Box setup

Storage Boxes are provisioned manually via the [Hetzner Robot panel](https://robot.hetzner.com/) — there is no Terraform resource for them.

### Steps

1. **Order** a Storage Box (BX11 or larger). Choose a Hetzner datacenter that is geographically separate from your primary infrastructure.

2. **Enable SSH support** in the Robot panel under the storage box settings. Set a password or use the web UI to manage SSH keys.

3. **Create the backup directory** for each host that will back up to this box:

   ```bash
   ssh -p 23 u123456@u123456.your-storagebox.de mkdir -p backups/hostname
   ```

4. **Register the SSH public key** — obtained by running the `backups-borg-client` role (it prints the key) or from `/root/.ssh/borg_ed25519.pub` on the host. Add it in the Robot panel under *Storage Box → SSH Keys*, or via:

   ```bash
   ssh-copy-id -i /root/.ssh/borg_ed25519.pub -p 23 u123456@u123456.your-storagebox.de
   ```

5. **Run the role** — on first backup run the Borg repository is initialised automatically.

### Sub-accounts (optional)

If multiple hosts back up to a single Storage Box, consider creating a sub-account per host in the Robot panel. Each sub-account has its own SSH key and is restricted to its own directory. Set `borg_storage_box_user` to the sub-account username.

### Verifying host key fingerprints

After role deployment, verify the storage box host key fingerprint against what Hetzner publishes in their documentation before trusting the first backup.

## Retention

Default retention (configurable per host):

| Period | Kept |
|---|---|
| Daily | 7 |
| Weekly | 4 |
| Monthly | 6 |
| Yearly | 2 |

## Recovery

See the [role README](../roles/backups-borg-client/README.md) for Borg commands. General procedure:

1. On a working host (or a fresh install), install `borgbackup`.
2. Set `BORG_REPO`, `BORG_PASSPHRASE`, and `BORG_RSH` to point at the Storage Box.
3. List available archives: `borg list`
4. Mount the desired archive: `borg mount ::archive-name /mnt/restore`
5. Copy files out, then `borg umount /mnt/restore`.

The repository key is stored in the repository itself (repokey encryption). You only need the passphrase to decrypt it — keep this passphrase in a secure location outside of this repository (e.g. a password manager).

## Key material

| Secret | Location |
|---|---|
| Borg passphrase | `vault_borg_passphrase` in host_vars (Ansible Vault) |
| SSH private key | `/root/.ssh/borg_ed25519` on each host (generated by role) |
| Borg repo key | Embedded in the repository on the Storage Box |
