# 1Password Connect Composition — Design

**Date:** 2026-05-18
**Host:** raspberry-pi4-4gb-randolph
**Role:** composition-1password-connect

## Architecture

Two Docker containers share a named volume and a credentials file:

```
op-connect-api   (:8080) ←── Traefik (connect.<domain>)
op-connect-sync            (no external exposure)
        ↕ shared Docker volume: 1password-connect-data
        ↕ shared credentials: {{ composition_config }}/1password-credentials.json
```

`connect-sync` polls 1Password cloud and writes synced vault data to the shared volume. `connect-api` reads from the shared volume and serves the REST API. Only `connect-api` joins the Traefik Docker network and receives Traefik labels.

`1password-credentials.json` is templated from a vault var to `{{ composition_config }}/1password-credentials.json` with `mode: 0600` before containers start.

## Secrets

One vault var, stored in `inventory/group_vars/infra/vault_onepassword_connect.yaml`:

- `vault_onepassword_connect_credentials` — full JSON content of `1password-credentials.json`, generated by running `op connect server create` before the playbook runs.

No Connect token is stored in the role. Tokens are per-integration, created in 1Password and stored separately when wiring up consumers (e.g. Ansible lookups).

## Bootstrap Requirement

`1password-credentials.json` must be generated manually before first run:

```bash
op connect server create randolph --vaults <vault-name>
```

This outputs `1password-credentials.json`. Its content is then encrypted:

```bash
ansible-vault encrypt_string "$(cat 1password-credentials.json)" \
  --name 'vault_onepassword_connect_credentials'
```

Paste the result into `vault_onepassword_connect.yaml`.

## Traefik Integration

`connect-api` container gets standard Traefik labels with subdomain `connect`. Port 8080. TLS via letsencrypt certresolver, same as all other compositions.

## Data Flow (Cold Rebuild)

1. Ansible Vault decrypts `vault_onepassword_connect_credentials`
2. Role templates `1password-credentials.json` to `{{ composition_config }}/`
3. Docker Compose starts both containers
4. `connect-sync` authenticates with 1Password cloud using the credentials file and populates the shared volume
5. `connect-api` begins serving from the shared volume
6. Subsequent roles/playbooks can use `OP_CONNECT_HOST` + `OP_CONNECT_TOKEN` to read secrets
